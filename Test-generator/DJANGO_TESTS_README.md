# Running Django Tests with StreamzAI Test Generator

This document provides instructions on how to run the Django tests that were generated by the StreamzAI Test Generator.

## Prerequisites

1. Python 3.6 or higher
2. Django installed in your environment
3. Access to the Django project for which tests were generated

## Setup

1. Make sure your Django project is properly set up and can run on its own
2. Ensure that the test files listed in `test_generation_summary.json` are accessible

## Running the Tests

You can run the Django tests using the provided `run_django_tests.py` script:

```bash
python run_django_tests.py --django-project-path "C:\path\to\your\django\project"
```

### Command-line Arguments

- `--summary-file`: Path to the test generation summary JSON file (default: "generated_tests/test_generation_summary.json")
- `--django-project-path`: Path to the Django project root directory (required)
- `--output`: Output file for test results in JSON format (optional)
- `--verbose` or `-v`: Enable verbose output (optional)

### Example

```bash
python run_django_tests.py --django-project-path "C:\Users\kusha\Desktop\Classroom\Projects\Portfolio\portfolio" --output test_results.json
```

## Understanding Test Results

After running the tests, a summary will be displayed showing:

- Overall test status
- Number of test files processed
- Number of passed, failed, and error files
- Number of passed, failed, error, and skipped tests
- Total execution time

If you specified an output file, detailed test results will be saved in JSON format.

## Troubleshooting

### Common Issues

1. **Django project path not found**: Ensure the path to your Django project is correct
2. **manage.py not found**: Make sure the Django project path contains a manage.py file
3. **Test files not found**: Verify that the test files listed in the summary file exist at the specified locations

### Django Environment Issues

If you encounter issues with the Django environment, try:

1. Activating your virtual environment if you're using one
2. Ensuring Django is properly installed (`pip install django`)
3. Checking that your Django project runs correctly on its own

## Additional Options

For more advanced usage, you can also use the underlying `test_runner.py` directly:

```bash
python test_runner.py --django --django-project-path "C:\path\to\your\django\project" --test-summary-file "generated_tests/test_generation_summary.json"
```
